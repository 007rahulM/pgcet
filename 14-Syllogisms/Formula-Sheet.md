# Syllogisms — Complete Rule Sheet

> 🎯 **HOW TO USE:** Identify the statement types. Apply the corresponding rule to draw a valid conclusion.

---

## ⚡ THE FOUR STATEMENT TYPES

| Type | Form | Symbol | Diagram |
|------|------|--------|---------|
| **Universal Affirmative** | All A are B | A (All) | A fully inside B |
| **Universal Negative** | No A is B | E (No) | A and B completely separate |
| **Particular Affirmative** | Some A are B | I (Some) | A and B partially overlap |
| **Particular Negative** | Some A are not B | O (Some...not) | Part of A outside B |

---

## 📐 RULE BLOCK 1 — Direct Valid Conclusions

### Always Valid:
| Statement | Valid Conclusion |
|-----------|----------------|
| All A are B | Some A are B (particular from universal) |
| All A are B | Some B are A |
| No A is B | No B is A |
| No A is B | Some A are not B |
| Some A are B | Some B are A |

### Never Valid (without more info):
| Statement | Invalid conclusion |
|-----------|------------------|
| Some A are B | All A are B ✗ |
| Some A are not B | No A is B ✗ |
| All A are B | All B are A ✗ (unless both universal) |

→ **See examples: [Pattern1-Basic-Syllogisms.md](./Pattern1-Basic-Syllogisms.md)**

## 📐 RULE BLOCK 2 — Two-Statement Chains

### Standard Combinations (Memorize):

| Statement 1 | Statement 2 | Conclusion |
|-------------|-------------|------------|
| All A are B | All B are C | **All A are C** ✓ |
| All A are B | Some B are C | **Some A are C** ✓ |
| All A are B | No B is C | **No A is C** ✓ |
| Some A are B | All B are C | **Some A are C** ✓ |
| Some A are B | No B is C | **Some A are not C** ✓ |
| No A is B | All C are B | **No C is A** ✓ |
| No A is B | Some C are B | **Some C are not A** ✓ |
| Some A are B | Some B are C | **No definite conclusion** ✗ |
| All A are B | Some C are B | **No definite conclusion** ✗ |

### ⚠️ Key Rule:
> If **both statements are Particular (Some)**, **no conclusion** can be drawn.
> If **both are Negative**, **no conclusion** can be drawn.

→ **See examples: [Pattern1-Basic-Syllogisms.md](./Pattern1-Basic-Syllogisms.md)**

## 📐 RULE BLOCK 3 — Possibility Cases

### Question looks like:
- "Which conclusion **may possibly be true**?"
- "Is it **possible** that some A are C?"

### Rules:
- If a definite conclusion cannot be proved **true** OR **false**, it is **possible**
- "Some A are not B" → it's **possible** that "Some A are B" (not ruled out)
- "No A is B" → it's **NOT possible** that "Some A are B" (ruled out completely)

→ **See examples: [Pattern2-Possibility-Cases.md](./Pattern2-Possibility-Cases.md)**

## 📐 RULE BLOCK 4 — Complementary Pairs

### In "Either-or" type questions:
> If neither conclusion individually follows, BUT they are **complementary** (I + O type for same terms), then **either-or** follows.

| Complementary Pair |
|--------------------|
| "All A are B" + "Some A are not B" |
| "Some A are B" + "No A is B" |

→ **See examples: [Pattern1-Basic-Syllogisms.md](./Pattern1-Basic-Syllogisms.md)**
