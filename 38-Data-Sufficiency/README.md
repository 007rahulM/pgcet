# Data Sufficiency — Overview

In data sufficiency, you decide whether statement A alone, B alone, both, or neither are enough to answer the question.

## Patterns in This Folder

| File | Pattern | Recognition Clue |
|------|---------|------------------|
| [questions.md](./questions.md) | DS practice set | "Is data sufficient?" |
| [solutions.md](./solutions.md) | Sufficiency classification | A alone / B alone / both / neither |

## Standard DS Answer Framework

1. A alone sufficient, B alone not sufficient
2. B alone sufficient, A alone not sufficient
3. Both together sufficient, neither alone sufficient
4. Both alone sufficient
5. Even together not sufficient

## Solved Examples

1. Is P Q’s sister?  
   A: P is daughter of Q’s father.  
   B: P is female.  
   A alone: could be sister or Q herself (if Q female) → not sufficient.  
   B alone: not sufficient.  
   Together: P female + daughter of Q’s father still could be Q (if Q female), so not certain.  
   **Answer: Even together not sufficient**

2. What is code for "BOOK"?  
   A: In a code, B=2, O=15, K=11.  
   B: Letters are replaced by position numbers.  
   A alone gives all needed values for BOOK.  
   **Answer: A alone sufficient**
